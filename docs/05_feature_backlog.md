# 05 功能 Backlog

## 1. 文档目的

本文档用于记录所有功能想法、优先级、状态、依赖关系和开发顺序。

注意：

- Backlog 中出现的功能不代表一定会做。
- 只有被确认为 P0 的功能才属于第一版 MVP。
- P1、P2、P3 功能默认不进入第一版开发计划。
- 新功能必须先进入 Backlog，再决定是否进入 MVP。

## 2. 优先级定义

| 优先级 | 含义 |
|---|---|
| P0 | MVP 必须有。没有这个功能，软件无法成立 |
| P1 | 重要功能，但可以在 MVP 之后实现 |
| P2 | 体验优化，不影响核心功能闭环 |
| P3 | 远期想法，暂不考虑 |
| 未定 | 需要进一步讨论 |

## 3. 状态定义

| 状态 | 含义 |
|---|---|
| 待讨论 | 只是想法，还没有确认 |
| 已确认 | 已确认要做，但未开始 |
| 设计中 | 正在明确交互、数据或技术细节 |
| 待开发 | 需求已清楚，可以进入开发 |
| 开发中 | 正在实现 |
| 待测试 | 已实现，等待验证 |
| 已完成 | 已通过验收 |
| 已推迟 | 暂时不做 |
| 已放弃 | 明确不做 |

## 7. 功能详情模板

后续每个功能可以按照以下格式展开。

### F001 本地账号注册

#### 功能描述

用户可以在本地创建账号。

#### 用户价值

让不同账号拥有独立的本地资料库。

#### 触发场景

用户首次使用软件，或希望创建新的本地身份。

#### 用户操作流程

1. 用户打开登录 / 注册页。
2. 用户输入用户名和密码。
3. 软件校验格式。
4. 软件创建本地账号。

#### 输入

用户名、密码。

#### 输出

注册成功提示，或格式错误 / 用户名已存在提示。

#### 异常情况

用户名长度不符合要求、用户名字符不合法、密码长度不符合要求、密码字符不合法、用户名已存在。

#### 验收标准

合法账号可以注册成功；非法输入会被拒绝并给出明确提示。

#### 依赖功能

本地数据存储。

#### 优先级

P0。

#### 当前状态

已确认。

### F006 影评时间戳插入

#### 功能描述

用户在影片影评编辑器中输入时、分、秒，点击添加后，软件在当前光标处插入统一格式时间戳。

#### 用户价值

降低记录影片具体时间点的操作成本。

#### 触发场景

用户撰写影评时想标记某个值得回看的位置。

#### 输入

时、分、秒。

#### 输出

影评文本中的时间戳。

#### 异常情况

输入为空、不是数字、超出合理范围、当前没有聚焦影评编辑器。

#### 验收标准

有效输入可以插入时间戳；无效输入不会污染影评文本，并提示用户修正。

#### 依赖功能

影片影评管理。

#### 优先级

P0。

#### 当前状态

已确认。

| 大模型推荐 | 本地资料库、API Key 配置、首页推荐提示词 | 配置有效时默认启用，首页文本框调用模型推荐影片和女优 |
| 大模型自动翻译 | 大模型 API 设置、翻译提示词、影片标题/简介字段 | 翻译启用后自动调用模型，并缓存翻译结果 |
| API Key 本地加密保存 | 大模型 API 设置、本地设置存储 | API Key 不能明文保存 |
| 影片详情重新翻译 / 取消翻译 | 大模型自动翻译、影片原始标题/简介字段 | 重新翻译会覆盖中文字段但保留原文字段；取消翻译回到原文显示 |

| 导出包结构 | 数据导入导出、图片本地缓存、可选加密导出 | 普通导出和加密导出都需要覆盖结构化数据、图片资源、manifest 和校验信息 |
| 页面草图 | 产品规格、主要页面清单 | 编码前先明确首页、库页面、详情页和设置页的主要布局 |

## 9. 开发顺序建议

初步建议：

1. 本地账号注册、登录和账号隔离。
2. 本地数据存储基础。
3. 影片资料和影评管理。
4. 女优资料管理。
5. 影片与女优关联。
6. 时间戳插入。
7. 自动保存。
8. 数据导入导出、默认普通导出和可选加密导出。
9. 影片和女优元数据匹配。
10. 图片本地缓存。
11. 元数据候选确认和手动编辑兜底。
12. 删除二次确认。
13. 作品类型与女优关联规则。
14. 标签库、预置标签、同义/关联标签检索、组合搜索、排序和筛选。
15. 自动年龄标签和自动 Cup 标签。
16. 女优多语种姓名匹配、显示名和罩杯筛选。
17. 中文标题和中文简介处理方案。
18. 大模型 API 设置、提示词设置和 API Key 加密保存。
19. 大模型首页推荐和标题/简介自动翻译。
20. 首页轮播和首页对话框样式。
21. MVP 验收后再评估安装包发布。

原则：

1. 先做核心闭环。
2. 先做可手动验证的功能。
3. 先做最小可运行版本。
4. 后做美化和体验优化。
5. 后做复杂配置。
6. 后做自动化和高级能力。
