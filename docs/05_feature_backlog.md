# 05 功能 Backlog

## 1. 文档目的

本文档用于记录所有功能想法、优先级、状态、依赖关系和开发顺序。

注意：

- Backlog 中出现的功能不代表一定会做。
- 只有被确认为 P0 的功能才属于第一版 MVP。
- P1、P2、P3 功能默认不进入第一版开发计划。
- 新功能必须先进入 Backlog，再决定是否进入 MVP。

## 2. 优先级定义

| 优先级 | 含义 |
|---|---|
| P0 | MVP 必须有。没有这个功能，软件无法成立 |
| P1 | 重要功能，但可以在 MVP 之后实现 |
| P2 | 体验优化，不影响核心功能闭环 |
| P3 | 远期想法，暂不考虑 |
| 未定 | 需要进一步讨论 |

## 3. 状态定义

| 状态 | 含义 |
|---|---|
| 待讨论 | 只是想法，还没有确认 |
| 已确认 | 已确认要做，但未开始 |
| 设计中 | 正在明确交互、数据或技术细节 |
| 待开发 | 需求已清楚，可以进入开发 |
| 开发中 | 正在实现 |
| 待测试 | 已实现，等待验证 |
| 已完成 | 已通过验收 |
| 已推迟 | 暂时不做 |
| 已放弃 | 明确不做 |
