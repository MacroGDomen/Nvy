当前任务只维护文档，不进入业务代码实现。

## 3. 全局强制规则

### 3.1 视觉风格

- UI 整体不能像上个世纪 Windows XP 资料管理系统。
- UI 整体参考 Google Gemini 和 Google AI Studio 暗夜模式。
- 设计风格偏 Google 风。
- 拿不准时采用 Material Design 3，但 Material Design 3 只作为兜底风格，不得覆盖用户的明确描述、手绘草图和给出的具体参考示例。
- 具体样式必须优先完全按照用户描述或用户给出的例子执行。
- 当用户描述、手绘草图、参考截图和 Material Design 3 发生冲突时，优先级为：用户明确文字描述 > 用户手绘草图 > 用户指定参考截图 / 参考源码 > Material Design 3 兜底规则。
- 不能用“Material Design 3”作为理由，把用户明确要求的布局、比例、位置、轮播形态、搜索框形态、下拉栏形态或页面边界改成通用后台式设计。
- 主色方向是淡淡的暗紫色、淡淡的暗粉色、渐变和暗灰背景。
- 背景为暗灰色。
- 搜索框、输入框、对话框、下拉框、按钮不能使用突兀白底。
- 允许使用柔和暗色渐变，但不能变成廉价、高饱和、大面积彩色渐变。
- 界面应有情趣、氛围和现代感，同时仍保持资料库软件的清晰可用。

### 3.2 16:9 强制比例

- 取消页面自由缩放。
- 软件主内容必须强制保持 16:9。
- 页面只能按 16:9 比例缩放。
- 当窗口按比例放大或缩小时，页面内部组件的相对位置不能漂移。
- 组件位置必须固定在 16:9 设计坐标中。
- 当页面整体缩放时，组件也按比例缩放。
- 组件内部的文字、按钮、图标也按比例缩放。
- 不允许出现窗口变大后组件之间相对位置改变、分布散掉、布局重新流动到另一种结构的情况。
- 实现时应优先采用一个固定 16:9 设计舞台，再按窗口可用空间进行整体等比例缩放。
- 16:9 舞台外的剩余空间可使用暗色背景填充，但不能影响舞台内部布局。

### 3.3 页面标题和分割线

- 首页最上方不要出现“首页”字样。
- 首页顶部不要有分割线。
- 影片库最上方不要出现“影片库”字样。
- 影片库顶部不要用页面标题占位。
- 所有页面都应避免无意义的大标题、说明文字和分割线。
- 页面内容应主要由草图中的核心组件构成，而不是传统后台管理页的标题 + 分割线 + 卡片堆叠结构。

### 3.4 菜单栏全局规则

- 本次重做 UI 时，菜单栏不用大改。
- 菜单栏不用按照用户手绘草图里的粗略菜单栏画法来实现。
- 菜单栏只做必要修正：滚动时固定、tooltip 出现在右侧、展开 / 折叠时 logo / 图标 / 头像固定大小和固定位置。
- 不要为了重做其他页面而重构菜单栏整体结构。
- 左侧菜单栏不按照手绘草图中粗略画法实现，实际以 Gemini 左侧状态栏、A01-A04 的交互为准。
- 菜单栏必须固定在视口左侧。
- 页面向下滚动时，菜单栏不能跟着滚动。
- 菜单栏折叠状态下不要太宽，宽度接近 Google Gemini 左侧图标状态栏。
- 默认折叠状态只显示图标，不显示中文文字。
- 首页、女优库、影片库、设置使用图标代表。
- 设置和用户头像固定放在菜单栏底部。
- 左上角显示 `NVY` logo 字样，风格参考 `logo参考.md`，但不要照抄 ASCII 块，只提取粗、几何、品牌字样的感觉。
- 当前选中的图标按钮需要有选中状态，参考 `A01.png` 的浅色提示底。
- 鼠标悬停到菜单按钮时显示 tooltip。
- tooltip 必须出现在菜单栏右侧。
- tooltip 不允许出现在按钮上方或下方。
- 鼠标悬停到 logo 时，logo 状态变化，参考 `A03.png`。
- 点击 logo 或展开按钮可以展开侧栏。
- 展开后显示按钮中文名称和账户名，参考 `A04.png`。
- 展开后需要有按钮可以收起状态栏。
- 展开和收起需要平滑动效。
- 展开前后，logo 大小固定。
- 展开前后，logo 位置固定。
- 展开前后，图标大小固定。
- 展开前后，图标位置固定。
- 展开前后，用户头像大小固定。
- 展开前后，用户头像位置固定。
- 展开前后，头像不能变大、变小、上移、下移或横向错位。
- 展开状态下出现的文字应在图标右侧展开，不应推动图标原有位置。
- 展开状态下账户名应出现在头像右侧或附近，但不能改变头像自身位置。

- 首页中“女优轮播”的字样全部去掉。
- 首页中“影片轮播”的字样全部去掉。
- 首页需要真正的轮播效果。
- 首页需要弧线。
- 首页轮播需要中间大、两侧小。
- 首页对话框上面不要有多余文字。
- 首页对话框样式参考 AI Studio 截图。
- AI Studio 的搜索框上面没有额外文字，因此 Nvy 首页对话框上面也不能放说明、标题或多余提示。
- 当前搜索框和 AI Studio 搜索框差异太大，需要重做。
- 首页取消自由缩放。
- 首页只能按 16:9 比例缩放。
- 首页强制页面为 16:9。
- 首页组件位置固定。
- 首页窗口按照比例放大时，组件也按照比例放大。
- 首页中的字和按钮也要等比例放大。
- 首页左上角 logo 是 `NVY` 字样。
- logo 风格参考 `logo参考.md`。
- 首页左侧竖菜单不要太宽。
- 首页左侧竖菜单宽度接近 Gemini 左侧状态栏。
- 首页菜单默认不要用文字。
- 首页菜单默认用像 Gemini 一样的图标代表选项。
- 菜单图标按下时，要像 `A01.png` 一样有颜色提示。
- 鼠标移动到按钮上时，要有像 `A02.png` 一样的提示。
- 头像和设置放在菜单栏最下面。
- 鼠标移动到 logo 上时，logo 要变化。
- 点按 logo 要可以展开侧边栏。

- 侧边栏需要有一个按钮可以收起。
- 首页女优头像轮播和影片封面轮播要像手绘草图一样。
- 首页轮播沿向下凹的曲线排布。
- 首页轮播每隔 3 秒自动顺时针轮播，视觉表现为当前项目整体向右侧移动。
- 女优头像下面要显示默认名字。
- 影片封面底下要显示 `番号 + 空格 + 名字`。
- 影片封面底下文字不要太长。
- 如果影片文字超过影片封面的长边宽度，用 `......` 代替。
- 轮播到最右边的影片或女优，在下一秒继续向软件右侧边缘移动，同时逐渐变小、渐隐并从右侧消失。
- 轮播最左侧的新项目必须从左侧边缘渐渐浮现，移动到第 1 个位置，同时逐渐变大到最小可见尺寸。
- 中间项目不能突然跳位；所有可见项目必须在上一位置和下一位置之间平滑移动。
- 女优轮播和影片轮播都必须多渲染左右两侧的过渡项目，不能只渲染当前可见项后直接替换。
- 首页下方对话框参考 Google AI Studio 的对话框设计。
- 女优和封面要可点击。

## 5. 页面结构描述

### 5.1 首页结构

#### 5.1.1 结构来源

首页结构以 `首页.jpg` 为主要布局来源，以 Google Gemini / Google AI Studio 暗色模式作为视觉风格来源。

#### 5.1.2 页面舞台

- 首页在 16:9 固定比例舞台内布局。
- 左侧是固定菜单栏。
- 右侧是首页主舞台。
- 主舞台不使用传统页面标题。
- 主舞台顶部不放“首页”文字。
- 主舞台顶部不放分割线。
- 主舞台内元素位置按草图固定，随 16:9 舞台等比例缩放。

#### 5.1.3 轮播区域

- 首页上半部分是轮播展示区。
- 女优头像轮播和影片封面轮播都在首页上半部分。
- 女优头像使用圆形视觉。
- 影片封面使用横向或竖向封面卡片，具体比例按影片封面资源决定，但视觉上要和草图中的矩形封面对应。
- 女优头像和影片封面沿向下凹的弧线分布。
- 弧线不是装饰线，而是轮播项的布局轨迹。
- 中间项最大。
- 中间项是视觉焦点。
- 两侧项逐步变小。
- 越靠近边缘越小、越弱、透明度可更低。
- 轮播每 3 秒自动顺时针推进一次，视觉表现为所有项目整体向右移动一个位置。
- 最右侧即将离场的项目在下一次推进时继续向软件右侧边缘移动、变小、渐隐并从右侧消失。
- 新项目从左侧边缘进入，逐渐浮现并变大到第 1 个可见位置的尺寸。
- 轮播不能突然替换数组导致抽动；必须保持过渡项连续移动。
- 影片轮播层级必须保证左侧靠近中心的项目压在右侧靠近中心的项目之上，避免从左往右第 4 部影片遮住第 3 部影片。
- 不允许把轮播做成普通水平列表。
- 不允许只做静态卡片网格。
- 不允许出现“女优轮播”文字。
- 不允许出现“影片轮播”文字。
- 女优头像下方显示默认名字。
- 影片封面下方显示 `番号 + 空格 + 名字`。
- 影片文字不得超过封面长边宽度。
- 影片文字超过限制时使用 `......`。
- 女优头像可点击。
- 影片封面可点击。
- 点击女优进入独立女优详情页。
- 点击影片进入独立影片详情页。

#### 5.1.4 AI 对话框区域

- 首页下方是 AI 推荐对话框。
- 对话框位置接近草图底部的大输入区域。
- 对话框参考 Google AI Studio 的底部输入框，不参考传统搜索栏。
- 对话框本体为暗灰色圆角输入容器。
- 对话框不使用白底。
- 对话框上方不放多余文字。
- 对话框上方不放“推荐”“提示”“输入偏好”等标题。
- 对话框上方不放解释性说明。
- 对话框内部可以有 placeholder，用于提示用户输入偏好、题材、标签或想看的类型。
- 对话框右侧或内部可放发送按钮、模型状态、麦克风 / 附加操作等小按钮，但必须低干扰。
- API Key 缺失等状态提示应低调处理，不能破坏首页草图结构。

### 5.2 菜单栏结构

#### 5.2.1 折叠状态

- 菜单栏固定在 16:9 舞台左侧。
- 菜单栏不随页面内容滚动。
- 顶部是 `NVY` logo。
- logo 下方是主要导航图标。
- 首页、女优库、影片库、设置都有独立图标。
- 设置和用户头像位于底部区域。
- 所有图标位置在折叠状态下固定。
- 当前页面图标有选中底色或选中提示。
- 鼠标悬停图标时，在右侧出现 tooltip。

#### 5.2.2 展开状态

- 展开时菜单栏宽度增加。
- 展开时图标原始位置不变。
- 展开时 logo 原始位置不变。
- 展开时用户头像原始位置不变。
- 中文文字和账户名在图标或头像右侧出现。
- 中文文字出现不能挤压图标。
- 头像大小不能变化。
- 展开 / 收起按钮位置固定。
- 展开 / 收起动效平滑，但不能导致元素跳动。

#### 5.3.1 结构来源

- 搜索按钮使用 Nvy 暗色 Google 风，不直接照抄蓝紫高亮。

- 顶部搜索栏不随内容滚动。

#### 5.3.4 影片网格

- 影片内容区域位于顶部搜索栏下方。
- 影片以 3 列、3 行显示，一屏 9 个封面。
- 封面之间保持草图中的横向留白。
- 封面尺寸按 16:9 舞台等比例固定。
- 封面下方只显示两行文本。
- 文本内容为 `番号 + 空格 + 名字`。
- 文本每一行不超过封面宽度。
- 超过两行时用 `......` 截断。
- 影片网格连续滚动，不分页。
- 向上滚动时，接近顶部搜索栏的封面和文字需要渐隐，形成帘式滚动。

- 设置页保留大模型配置、API Key、接口类型、Base URL、模型型号 / 模型 ID、参数、提示词、推荐参考总数、标签库管理、导入导出、安全说明等 MVP 必要内容。
- 设置页输入框和下拉栏使用全局输入与下拉样式。

## 6. UI 修改步骤

### 6.1 第一阶段：建立全局布局基线

1. 开始实现前重新查看对应页面草图和对应控件参考图。
2. 明确本轮具体页面使用的参考优先级：用户文字描述优先，其次手绘草图，其次参考截图 / 参考源码，最后才是 Material Design 3 兜底。
3. 建立 16:9 固定比例应用舞台。
4. 让应用舞台根据窗口可用空间等比例缩放。
5. 禁止页面内部根据窗口自由重排。
6. 所有页面内容挂载到同一个 16:9 舞台坐标体系中。
7. 清理首页、影片库等页面顶部的大标题和分割线。

### 6.2 第二阶段：重做左侧菜单栏

1. 菜单栏本轮不用大改，不按用户草图里的粗略菜单栏重做。
2. 菜单栏只修正当前明确问题。
3. 菜单栏改为固定定位，不随页面滚动。
4. 折叠状态下只显示 logo、图标、底部设置和头像。
5. 展开状态下保持 logo、图标、头像大小和位置不变。
6. 展开状态只在图标右侧显示中文和账户名。
7. 修复展开 / 收起时图标、logo、头像位移和缩放问题。
8. tooltip 固定显示在菜单栏右侧。
9. tooltip 不再出现在按钮上方或下方。
10. 菜单栏选中、hover、展开、收起状态按 A01-A04 参考做必要修正。

### 6.3 第三阶段：重做首页

1. 删除首页顶部“首页”字样。
2. 删除首页顶部多余分割线。
3. 删除“女优轮播”和“影片轮播”文字。
4. 根据 `首页.jpg` 建立首页轮播布局。
5. 实现向下凹弧线轨迹。
6. 实现中间大、两侧小的视觉层级。
7. 实现每 3 秒自动顺时针轮播，项目整体向右移动。
8. 实现最右侧项目向右变小并渐隐离场，最左侧项目从左侧渐显入场。
9. 修正影片轮播层级，确保从左往右第 3 部影片显示在第 4 部影片上层。
10. 女优头像和影片封面支持点击进入详情页。
11. 按 Google AI Studio 底部输入框重做首页对话框。
12. 删除对话框上方多余文字。
13. 对话框内部只保留必要 placeholder、发送按钮和低干扰状态。

### 6.4 第四阶段：重做影片库

1. 确保影片库页面不渲染影片详情。
2. 影片详情改为独立路由 / 独立页面。

## 8. 已确认结论

- 已确认：整体 UI 参考 Google Gemini / Google AI Studio 暗夜模式。
- 已确认：拿不准时采用 Material Design 3的设计风格。
- 已确认：Material Design 3 只是兜底风格，不能覆盖用户明确描述、用户手绘草图或用户给出的具体参考例子。
- 已确认：具体样式必须优先完全按照用户描述和用户给出的例子执行。
- 已确认：开始写任意页面 UI 之前，必须重新查看该页面对应的用户手绘草图。
- 已确认：此次重做 UI 时，菜单栏不用大改。
- 已确认：菜单栏不用按照用户手绘草图里的菜单栏画法来。
- 已确认：菜单栏只做滚动固定、tooltip 右侧显示、展开 / 折叠时 logo / 图标 / 头像固定大小和位置等必要修正。
- 已确认：暗灰背景，淡暗紫和淡暗粉为主要强调方向。
- 已确认：首页顶部不显示“首页”字样。
- 已确认：首页顶部不显示分割线。
- 已确认：首页不显示“女优轮播”和“影片轮播”文字。
- 已确认：首页轮播要有向下凹弧线。
- 已确认：首页轮播要中间大、两侧小。
- 已确认：首页轮播每 3 秒自动顺时针轮播，项目整体向右移动。
- 已确认：首页轮播最右侧项目向右变小并渐隐离场，最左侧项目从左侧渐显入场。
- 已确认：影片轮播从左往右第 3 部影片应显示在第 4 部影片上层，不能被第 4 部遮住。
- 已确认：首页对话框参考 Google AI Studio 输入框。
- 已确认：首页对话框本体上方不放说明性标题、提示或多余文字；问候语可以作为独立页面文案显示在对话框上方。
- 已确认：全局页面强制 16:9，按比例整体缩放。
- 已确认：菜单栏滚动时固定。
- 已确认：菜单栏 tooltip 出现在右侧。
- 已确认：菜单栏展开 / 折叠时 logo、图标、头像固定大小和位置。
- 已确认：影片库不显示影片详情。
- 已确认：影片库和影片详情是不同页面。
- 已确认：点击影片才进入影片详情。
- 已确认：删除影片库顶部“影片库”文字。
- 已确认：影片库搜索栏按 `as23.png` 方向重做，暗灰、圆边框。
- 已确认：女优库搜索栏按 `as23.png` 方向重做，暗灰、圆边框。
- 已确认：影片库一屏 3×3 封面。
- 已确认：女优库一屏 3×5，共 15 个头像；结合草图按 5 列、3 行实现。
- 已确认：库页面连续滚动，不分页。
- 已确认：滚动内容接近固定搜索栏时渐隐。
- 已确认：下拉栏参考两张下拉截图。
