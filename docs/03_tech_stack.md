# 03 技术栈选择

## 1. 文档目的

本文档用于记录项目的技术栈选择、选择理由、备选方案、放弃原因、复杂度影响和风险。

技术栈不是越先进越好。本项目优先考虑：

1. 是否适合当前软件目标。
2. 是否适合第一版 MVP。
3. 是否适合项目创建者理解和维护。
4. 是否适合 Coding Agent 稳定生成和修改代码。
5. 是否有足够成熟的文档和社区。
6. 是否会引入不必要的复杂度。

## 3. 技术选择原则

### 3.1 优先原则

1. 优先简单。
2. 优先稳定。
3. 优先可维护。
4. 优先文档丰富。
5. 优先适合 AI Agent 修改。
6. 优先适合 MVP 快速验证。
7. 优先减少新手心智负担。

### 3.2 避免原则

1. 避免过早使用微服务。
2. 避免过早引入复杂云架构。
3. 避免过早引入复杂权限系统。
4. 避免为小项目引入大型框架组合。
5. 避免多个技术栈混用导致维护困难。
6. 避免没有明确理由的技术选择。
