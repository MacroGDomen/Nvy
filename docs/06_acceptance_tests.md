# 06 验收测试

## 1. 文档目的

本文档用于记录软件功能的验收标准。

验收测试的目标不是证明代码写得漂亮，而是证明软件行为符合用户需求。

每个核心功能都应该有明确的验收方式。

## 2. 验收原则

1. 验收标准必须具体。
2. 验收标准必须能被手动验证。
3. 验收标准必须和用户场景相关。
4. 不要只写“功能正常”。
5. 应该说明操作步骤、输入、期望结果。
6. 异常情况也需要验收。
7. MVP 功能必须有验收标准。

## 4. 核心流程验收

### A001 本地记录核心流程验收

#### 对应功能

F001-F010、F014-F017。

#### 前置条件

软件已安装或可运行，当前没有依赖服务器。

#### 操作步骤

1. 打开软件并选择离线使用。
2. 注册一个合法本地账号。
3. 使用该账号登录。
4. 新增一位女优并填写名字、三围、中文维基百科链接和个人评价。
5. 输入女优名并尝试匹配头像和基础资料。
6. 在候选结果中选择一个结果，或跳过匹配后手动录入。
7. 新增一部影片并输入番号。
8. 尝试匹配影片封面、标题和基础资料。
9. 在候选结果中选择一个结果，或跳过匹配后手动录入。
10. 将影片标记为单人作品。
11. 撰写影片影评。
12. 在影评中插入 `[01:23:45]` 格式时间戳。
13. 将单人作品关联到该女优。
14. 新增一部多人作品，并确认软件禁止添加或关联女优。
15. 确认匹配得到的封面和头像已缓存到本地。
16. 导出当前账号数据，并分别测试默认普通导出和可选加密导出。
17. 关闭软件后重新打开并登录同一账号。
18. 查看影片详情、女优详情和首页轮播。
19. 在测试环境中导入此前导出的数据并检查恢复结果。
20. 删除影片或女优时验证二次确认；删除女优后确认影片仍保留。

#### 期望结果

软件能显示此前保存的账号、影片、影评、时间戳、女优资料、匹配资料、手动修改内容和关联关系；导入后数据能够恢复。

#### 通过标准

不需要手动保存，重启后数据仍存在且归属正确账号；导出文件可用于恢复资料库。

#### 状态

未开始。

## 6. 异常情况验收

| 编号 | 异常情况 | 操作步骤 | 期望提示 / 行为 | 状态 |
|---|---|---|---|---|
| E001 | 用户名长度不合法 | 注册时输入少于 3 位或超过 10 位用户名 | 拒绝注册并提示用户名规则 | 未开始 |
| E002 | 用户名包含不支持字符 | 注册时输入字母、数字、中文以外字符 | 拒绝注册并提示用户名规则 | 未开始 |
| E003 | 密码长度不合法 | 注册时输入少于 3 位或超过 20 位密码 | 拒绝注册并提示密码规则 | 未开始 |
| E004 | 密码包含不支持字符 | 注册时输入字母、数字以外字符 | 拒绝注册并提示密码规则 | 未开始 |
| E005 | 用户名重复 | 使用已存在用户名注册 | 拒绝注册并提示用户名已存在 | 未开始 |
| E006 | 登录密码错误 | 输入错误密码登录 | 拒绝登录并提示账号或密码不正确 | 未开始 |
| E007 | 时间戳输入无效 | 输入空值、非数字或超出范围的时分秒 | 不插入时间戳，并提示格式错误 | 未开始 |
| E008 | 数据保存失败 | 模拟本地数据不可写 | 提示保存失败，并尽可能保留当前编辑内容 | 未开始 |
| E009 | 元数据匹配失败 | 输入不存在或无法访问的番号 / 女优名 | 提示未找到匹配结果，并允许手动录入或编辑 | 未开始 |
| E010 | 导入文件无效 | 选择损坏、格式错误或版本不兼容的导入文件 | 拒绝导入并提示原因，不破坏现有数据 | 未开始 |
| E011 | 加密导入密码错误 | 导入加密导出文件时输入错误密码 | 拒绝导入并提示密码不正确，不破坏现有数据 | 未开始 |
| E012 | 多人作品添加女优 | 将影片标记为多人作品后尝试添加或关联女优 | 拒绝操作并提示多人作品不添加女优 | 未开始 |
| E013 | 删除时取消确认 | 删除影片或女优时在二次确认中取消 | 不删除任何数据 | 未开始 |

需要考虑的异常情况：

- 用户没有输入必要信息。
- 用户输入格式错误。
- 用户取消操作。
- 数据保存失败。
- 网络断开。
- 权限不足。
- 文件不存在。
- 文件格式错误。
- 第三方服务不可用。
- 软件运行时发生异常。

## 7. 数据验收

需要重点验证：

- 账号数据是否正确保存。
- 密码是否不以明文方式保存。
- 影片、女优、影评和关联关系是否正确保存与读取。
- 影片作品类型是否正确限制女优关联。
- 删除女优后影片是否仍保留。
- 元数据匹配得到的封面、头像和资料字段是否可保存与修改。
- 匹配得到的封面和头像是否缓存到本地，并随导入导出保持完整。
- 不同账号之间数据是否隔离。
- 自动保存是否可靠。
- 导入导出功能需要验证数据完整性、账号归属和版本兼容。

## 8. 界面验收

第一版界面需要验证：

- 用户能找到登录、注册、影片、女优、设置入口。
- 时间戳输入和添加按钮位置清楚。
- 影评编辑区域不会因为插入时间戳导致光标错乱。
- 错误提示清楚，不暴露底层异常。
- 首页对话框在第一版只作为样式存在，不应误导用户认为推荐已可用。
- 首页轮播展示所有已记录影片和女优，点击跳转正确。

## 10. 安全与隐私验收

需要重点验证：

- 成人内容相关私人记录是否只保存在预期本地位置。
- 密码是否不会明文保存。
- 第一版数据库文件不做整体加密，该限制需要在安全说明中明确。
- 加密导出文件是否必须输入正确密码才能恢复。
- API Key 是否不会出现在日志或代码仓库中。
- 未登录状态下不能访问账号数据。
- 账号 A 不能访问账号 B 的影片、女优、影评和设置。
- 元数据匹配请求会发送番号或女优名，需要在产品说明中明确。
- 调用大模型 API 前，需要明确用户输入和本地资料中哪些内容会被发送；大模型实际推荐不进入 MVP。

## 11. 回归测试清单

每次重要修改后，至少检查：

- 核心流程是否还能完成。
- 已完成的 P0 功能是否仍然可用。
- 数据是否仍能正常保存和读取。
- 软件是否能正常启动。
- 新功能是否破坏旧功能。
- 文档是否需要同步更新。
