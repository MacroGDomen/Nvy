# 06 验收测试

## 1. 文档目的

本文档用于记录软件功能的验收标准。

验收测试的目标不是证明代码写得漂亮，而是证明软件行为符合用户需求。

每个核心功能都应该有明确的验收方式。

## 2. 验收原则

1. 验收标准必须具体。
2. 验收标准必须能被手动验证。
3. 验收标准必须和用户场景相关。
4. 不要只写“功能正常”。
5. 应该说明操作步骤、输入、期望结果。
6. 异常情况也需要验收。
7. MVP 功能必须有验收标准。

需要考虑的异常情况：

- 用户没有输入必要信息。

## 7. 数据验收

暂未确定。

需要验证：

- 数据是否正确保存。
- 数据是否正确读取。
- 数据是否会丢失。
- 数据格式是否稳定。
- 用户错误操作是否会破坏数据。
- 旧数据是否兼容新版本。

## 8. 界面验收

暂未确定。

需要验证：

- 用户是否能看懂界面。
- 主要按钮是否容易找到。
- 错误提示是否清楚。
- 页面状态是否明确。
- 加载中、成功、失败状态是否可见。
- 小屏幕或不同分辨率下是否可用。

## 9. 性能验收

暂未确定。

可能需要验证：

- 启动速度。
- 页面响应速度。
- 数据处理速度。
- 大量数据下是否卡顿。
- 长时间运行是否稳定。
- 内存占用是否异常。

## 10. 安全与隐私验收

暂未确定。

可能需要验证：

- 用户数据是否只保存在预期位置。
- 敏感信息是否不会明文暴露。
- 未授权用户是否无法访问受限内容。
- 第三方 API Key 是否不会提交到代码仓库。
- 用户删除数据后是否真的删除。

## 11. 回归测试清单

每次重要修改后，至少检查：

- 核心流程是否还能完成。
- 已完成的 P0 功能是否仍然可用。
- 数据是否仍能正常保存和读取。
- 软件是否能正常启动。
- 新功能是否破坏旧功能。
- 文档是否需要同步更新。
