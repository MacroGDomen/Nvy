## 14. 错误处理要求

暂未确定。

错误处理原则：

1. 不应该让用户看到难以理解的底层错误。
2. 软件应该提示用户发生了什么。
3. 软件应该提示用户可以怎么处理。
4. 关键数据不能因为错误而丢失。
5. 对可恢复错误，应提供重试或返回操作。
