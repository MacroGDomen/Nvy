# Nvy

## 1. 项目简介

本项目是一个正在设计中的 Windows 本地桌面软件。

当前产品方向是：为成年个人用户提供成人影片私人记录能力，用于保存影片资料、女优资料、个人影评、影片时间点感受，并通过用户自填 API 接入大模型进行个人资料库内的检索推荐和影片标题/简介翻译。

当前访谈阶段已结束。下一步应根据已确认文档撰写第一版 MVP 开发计划；在用户明确说“开始实现代码”前，不进入代码实现阶段。

项目采用“文档先行”的开发方式。

## 2. 当前状态

当前状态：需求访谈阶段已结束，下一步进入开发计划阶段。

暂不进入正式编码阶段。

第一版技术栈已确认：Tauri + React + TypeScript + SQLite。

第一版 UI 方向已调整为参考 Google Gemini / Google AI Studio 暗夜模式，采用 Material Design 3 倾向、暗灰背景、淡暗紫和淡暗粉强调色。

## 3. 项目目标

第一版目标是先完成本地离线核心闭环：

- 本地账号注册和登录。
- 多个本地账号之间的数据隔离。
- 通过番号和女优名匹配影片封面、女优头像和女优资料，匹配结果可在软件内自由修改。
- 匹配成功后的封面和头像缓存到本地。
- 影片资料、女优资料、影评和时间戳记录。
- 影评时间戳格式为 `[01:23:45]`。
- 作品类型包含单人作品、多人作品、素人；番号和作品类型必填，单人作品女优必填。
- 多人作品只自动关联已收藏女优，未收藏女优不自动创建，后续添加匹配女优时弹出确认后补关联。
- 素人作品不强制填写女优，但允许手动关联已收藏女优。
- 删除影片或女优需要二次确认，确认后永久删除；删除女优不删除影片。
- 影片和女优有独立标签库，支持预置标签、组合搜索、同义词、繁简归一和关联标签检索。
- 影片标题和简介优先中文显示，第一版使用大模型自动翻译并缓存翻译结果。
- 设置页包含“大模型设置”，支持 API Key 本地加密保存、接口类型切换、Base URL、模型型号 / 模型 ID、模型参数、推荐参考总数上限、翻译提示词和首页推荐提示词配置。
- 第一版首页下方文本框实际调用大模型，与用户交互并检索本地数据库，推荐影片和女优；推荐默认启用。
- 第一版调用大模型自动翻译影片标题和简介，翻译结果保存到本地，避免重复翻译。
- 数据导入导出，默认普通导出，支持可选加密导出。
- 第一版不整体加密数据库，但本地账号密码必须哈希保存，并保留后续整体加密升级可能。
- 首页轮播展示所有已记录影片和女优，加入软件即视为收藏。
- 本地自动保存。
- 后续查看和回顾记录。

第一版暂不做在线服务器同步、云端账号、成人影片播放或下载。第一版先供项目创建者本人使用，稳定后再考虑安装包。

## 6. 工作方式

本项目的推荐工作流程是：

1. 用户描述软件想法。
2. Coding Agent 提出澄清问题。
3. 用户回答问题。
4. Coding Agent 更新文档。
5. 用户审查文档。
6. 文档稳定后再制定开发计划。
7. 用户确认后再开始写代码。
8. 每次代码变更后同步更新相关文档。

## 7. 当前不做的事情

在项目早期，暂时不做：

- 不写业务代码。
- 不搭建复杂工程。
- 不引入未经确认的框架。
- 不实现用户没有确认的功能。
- 不把临时想法直接当成正式需求。
- 不提供成人影片播放、下载或分发能力。
- 不实现在线服务器同步。

## 8. 如何继续推进

下一步应该推进以下事项：

1. 根据已确认的 MVP 文档拆分开发计划。
2. 在用户明确说“开始实现代码”后，再进入工程创建和实现阶段。
