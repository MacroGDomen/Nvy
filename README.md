# Nvy

## 1. 项目简介

本项目是一个正在设计中的 Windows 本地桌面软件。

当前产品方向是：为成年个人用户提供成人影片私人记录能力，用于保存影片资料、女优资料、个人影评、影片时间点感受，并在后续通过用户自填 API 接入大模型进行个人资料库内的检索和推荐。

当前阶段的重点不是立即写代码，而是通过和 Coding Agent 对话，逐步明确软件目标、目标用户、核心功能、技术栈和验收标准。

项目采用“文档先行”的开发方式。

## 2. 当前状态

当前状态：需求理解与文档维护阶段。

暂不进入正式编码阶段。

## 3. 项目目标

第一版目标是先完成本地离线核心闭环：

- 本地账号注册和登录。
- 多个本地账号之间的数据隔离。
- 通过番号和女优名匹配影片封面、女优头像和女优资料，匹配结果可在软件内自由修改。
- 匹配成功后的封面和头像缓存到本地。
- 影片资料、女优资料、影评和时间戳记录。
- 影评时间戳格式为 `[01:23:45]`。
- 单人作品可添加并关联女优；多人作品禁止添加或关联女优。
- 删除影片或女优需要二次确认；删除女优不删除影片。
- 数据导入导出，默认普通导出，支持可选加密导出。
- 第一版不整体加密数据库，但本地账号密码必须哈希保存，并保留后续整体加密升级可能。
- 首页轮播展示所有已记录影片和女优，加入软件即视为收藏。
- 本地自动保存。
- 后续查看和回顾记录。

第一版暂不做在线服务器同步、云端账号、成人影片播放或下载。第一版也不实现大模型推荐，只保留首页对话框样式。第一版先供项目创建者本人使用，稳定后再考虑安装包。

## 6. 工作方式

本项目的推荐工作流程是：

1. 用户描述软件想法。
2. Coding Agent 提出澄清问题。
3. 用户回答问题。
4. Coding Agent 更新文档。
5. 用户审查文档。
6. 文档稳定后再制定开发计划。
7. 用户确认后再开始写代码。
8. 每次代码变更后同步更新相关文档。

## 7. 当前不做的事情

在项目早期，暂时不做：

- 不写业务代码。
- 不搭建复杂工程。
- 不引入未经确认的框架。
- 不实现用户没有确认的功能。
- 不把临时想法直接当成正式需求。
- 不提供成人影片播放、下载或分发能力。
- 不实现在线服务器同步。
