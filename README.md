## 6. 工作方式

本项目的推荐工作流程是：

1. 用户描述软件想法。
2. Coding Agent 提出澄清问题。
3. 用户回答问题。
4. Coding Agent 更新文档。
5. 用户审查文档。
6. 文档稳定后再制定开发计划。
7. 用户确认后再开始写代码。
8. 每次代码变更后同步更新相关文档。

## 7. 当前不做的事情

在项目早期，暂时不做：

- 不写业务代码。
- 不搭建复杂工程。
- 不引入未经确认的框架。
- 不实现用户没有确认的功能。
- 不把临时想法直接当成正式需求。
